Construyendo un Portafolio Serverless en AWS con Terraform

El objetivo principal fue crear un portafolio personal moderno, rápido, seguro, escalable y eficiente en costos, utilizando servicios serverless de AWS y gestionado como Infraestructura como Código (IaC).

Frontend

  • Base: Plantilla iPortfolio (BootstrapMade) con Bootstrap 5.
  • Tecnologías: HTML, CSS, JavaScript.
  • Características:
    • ✔ Diseño responsive
    • ✔ Internacionalización (i18n) con JavaScript
    • ✔ Carga dinámica del header compartido
    • ✔ Envío de formulario de contacto vía API
  • Despliegue: Hosting estático en AWS S3. Actualmente manual con aws s3 sync. Próximo paso: automatización con CI/CD (GitHub Actions o AWS CodePipeline).
  • Repositorio: github.com/xavierux/iPortfolio

Infraestructura y Backend

  • Cloud: AWS.
  • Arquitectura Backend: API RESTful serverless para gestionar el formulario de contacto.
  • Servicios AWS: API Gateway (HTTP API), AWS Lambda, AWS SES.
  • Lenguaje Lambda: Python.
  • Funcionamiento Backend: Recibe datos JSON del frontend, valida, compone y envía emails mediante SES.
  • IaC: Terraform para definir y gestionar recursos AWS (S3, CloudFront, OAC, Lambda, API Gateway, IAM Roles/Políticas, SES Identity/Permissions).
  • CDN y Seguridad:
    • ✔ CloudFront para distribución global
    • ✔ HTTPS automático con certificado por defecto
    • ✔ Acceso seguro a S3 mediante Origin Access Control (OAC)
  • Repositorio: github.com/xavierux/iPortfolio-Infra
